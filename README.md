# Voice AI Receptionist - Rose and Bloom Flowers

A fully functional AI voice receptionist named **Lily** built for Rose and Bloom Flowers, a fictitious flower shop in Grand Rapids, Michigan. Lily answers incoming calls, handles common customer questions, collects caller information, and emails the business owner a detailed summary after every call.

Built by **Joyce Malicha** as a portfolio project demonstrating real-world AI integration with voice, language models, and business automation.

---

## Live Demo Flow

1. A customer calls the business phone number
2. Vapi.ai answers the call and handles all voice processing (speech to text and text to speech)
3. Lily (powered by Claude AI) responds naturally and conversationally
4. Lily answers questions about hours, services, and pricing
5. If she does not know something, she collects the caller's name and number for a follow-up
6. When the call ends, the business owner receives a detailed email summary with caller name, contact number, what they needed, and any action required

## How It Works

### Webhook Flow

Vapi sends a POST request to `/webhook` for each conversation event. The server processes `end-of-call-report` events and sends an email summary to the business owner via Resend.

### Email Summary

After every call, the owner receives an email with:
- Caller name (extracted from conversation)
- Caller phone number (caller ID and any number they gave verbally)
- Full call summary generated by Vapi's AI analysis
- Action needed (callback, consultation, order, or none)

### Lily's Personality

Lily is configured via `prompts/system_prompt.txt`. She:
- Keeps every response to one or two sentences
- Never makes up prices or availability she was not given
- Always collects caller info when she cannot answer a question
- Ends every call with a warm goodbye before hanging up

## Setup Instructions

### 1. Clone the repository

```bash
git clone https://github.com/jmalicha/voice-ai-receptionist.git
cd voice-ai-receptionist
```

### 2. Create and activate a virtual environment

```bash
python3 -m venv venv
source venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Set up environment variables

Copy the example file and fill in your credentials:

```bash
cp .env.example .env
```

Open `.env` and add your keys:

```
ANTHROPIC_API_KEY=your_anthropic_api_key
VAPI_API_KEY=your_vapi_api_key
TWILIO_ACCOUNT_SID=your_twilio_account_sid
TWILIO_AUTH_TOKEN=your_twilio_auth_token
TWILIO_PHONE_NUMBER=your_twilio_phone_number
OWNER_PHONE_NUMBER=owner_phone_number
RESEND_API_KEY=your_resend_api_key
OWNER_EMAIL=owner_email_address
```

### 5. Run the server

```bash
uvicorn main:app --reload --port 8000
```

### 6. Expose your server with ngrok

In a second terminal:

```bash
ngrok http 8000
```

Copy the HTTPS URL (e.g. `https://abc123.ngrok-free.app`)

## Connecting to Vapi

1. Log in to [vapi.ai](https://vapi.ai)
2. Create an assistant with the Anthropic provider using the system prompt from `prompts/system_prompt.txt`
3. Set the assistant server URL to: `https://your-ngrok-url/call-ended`
4. Set end call phrases: `bye bye`, `goodbye`, `have a wonderful day`
5. Connect your Twilio phone number to the assistant
6. Call your number and Lily will answer

## Environment Variables

| Variable | Description |
|---|---|
| `ANTHROPIC_API_KEY` | Anthropic API key for Claude |
| `VAPI_API_KEY` | Vapi.ai API key |
| `TWILIO_ACCOUNT_SID` | Twilio account SID |
| `TWILIO_AUTH_TOKEN` | Twilio auth token |
| `TWILIO_PHONE_NUMBER` | Twilio phone number (E.164 format) |
| `OWNER_PHONE_NUMBER` | Business owner's phone number |
| `RESEND_API_KEY` | Resend API key for email notifications |
| `OWNER_EMAIL` | Email address to receive call summaries |
